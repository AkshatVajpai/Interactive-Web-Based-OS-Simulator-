FIFO

FIFO (First In First Out)

The simplest scheduling algorithm where processes are executed in the order they arrive. Like a queue at a ticket counter, the first process to arrive is the first to be executed.

Key Characteristics:

  • Simple to implement
  • No starvation
  • Poor average waiting time
  • Non-preemptive
SJF

SJF (Shortest Job First)

Executes processes with the shortest burst time first. Like a smart queue where shorter tasks are prioritized, minimizing the average waiting time.

Key Characteristics:

  • Minimizes average waiting time
  • Optimal for batch systems
  • Requires knowledge of process lengths
  • Can cause starvation
PRI

Priority Scheduling

Processes are executed based on their priority levels. Like a hospital emergency room where critical patients are treated first, regardless of arrival time.

Key Characteristics:

  • Processes with higher priority execute first
  • Useful for real-time systems
  • Can be preemptive or non-preemptive
  • May cause starvation
RR

Round Robin

Each process gets a fixed time quantum for execution. Like a time-sharing system where each process gets an equal slice of CPU time.

Key Characteristics:

  • Fair allocation of CPU time
  • Prevents starvation
  • Good for time-sharing systems
  • Performance depends on time quantum
